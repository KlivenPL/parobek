# F3.2 — Digest tools

- [ ] **Status:** not started
- **Tier:** 1 (Safe) · **Phase:** NOW · **Epic:** 3

## Goal

The mechanical tool set the MCP server exposes. Each is pure compression / retrieval /
extraction — **no judgment**, so Anthropic stays the brain and only offloads the
grunt work.

## Tools

| Tool | Signature | What it does |
|------|-----------|--------------|
| `local_summarize` | `(text \| path)` | Condense a big blob (log, file, diff, web dump) to a digest. Pure compression. |
| `local_read_digest` | `(path, question?)` | Return only the relevant slice/structure of a file. Retrieval/filtering. |
| `local_extract` | `(text, schema)` | Mechanical structured extraction (error codes, endpoints, TODOs → JSON). |
| `local_grep_digest` | `(pattern, path?)` | Run ripgrep, then digest the hits so Claude doesn't read hundreds of match lines. |
| `local_outline` | `(path)` | Structural outline (symbols/headings) without the full body. |
| `local_log_triage` | `(path)` | Condense a huge log, keeping stack traces/errors **verbatim**, dropping noise. |
| `local_diff_digest` | `(ref?)` | Per-file bullet summary of a large `git diff`. |

## Mechanism

- Each tool = a registry entry (F3.1) with a JSON input schema + a handler that builds
  messages and calls `provider.chat`.
- Prompts live in a `lib/digest-prompt.mjs` family (one focused prompt per tool),
  following the prompt-module pattern of
  [`compact-prompt.mjs`](../../plugins/parobek/scripts/lib/compact-prompt.mjs).
- Path-taking tools read the file (reuse `transcript.mjs` truncation for oversize
  inputs) and fall back to map-reduce (reuse `tokens.mjs` `chunkMessages`) when the
  input exceeds the local budget — so large inputs are non-fatal.
- `local_grep_digest` shells out to ripgrep (already a Claude Code dependency) and
  digests the output. `local_diff_digest` shells out to `git diff`.
- `local_extract` returns JSON matching the caller-supplied schema; on parse failure
  it returns the raw text + an error note (never throws).

## Files

- **Add:** `lib/digest-prompt.mjs`, tool handlers (in `mcp/server.mjs` or
  `mcp/tools/*.mjs`).
- **Reuse:** `lib/provider.mjs`, `lib/transcript.mjs`, `lib/tokens.mjs`,
  `lib/brand.mjs`.

## Feasibility / risk

- All tier 1: Anthropic decides when to call and can always raw-read instead. No
  silent substitution.
- Keep handlers strictly mechanical — if a tool starts "deciding," it has drifted out
  of tier 1.

## Acceptance & verification

- Per-tool integration against `mock-server.mjs`: assert each returns the expected
  shape (`local_extract` → valid JSON; `local_outline` → headings; etc.).
- Assert oversize inputs trigger the map-reduce path and still return a digest.
