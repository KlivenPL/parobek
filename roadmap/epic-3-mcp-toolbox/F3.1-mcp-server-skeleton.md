# F3.1 — MCP server skeleton

- [x] **Status:** done
- **Tier:** 1 (Safe) · **Phase:** NOW · **Epic:** 3

## Goal

A working MCP stdio server that Claude Code can launch, advertising the plugin's local
tools — built with **zero runtime dependencies**.

## Mechanism

- `mcp/server.mjs`: a minimal hand-rolled **JSON-RPC 2.0 over stdio** loop.
  - Read line-delimited (or `Content-Length`-framed) JSON from stdin; write responses
    to stdout. Pick the framing Claude Code's MCP client expects (see the MCP spec and
    the official Claude Code docs at https://code.claude.com/docs).
  - Handle the three methods needed: `initialize` (return server info +
    capabilities), `tools/list` (return the tool schemas), `tools/call` (dispatch by
    name to a handler, return `content` blocks).
  - A simple tool registry: `{ name, description, inputSchema, handler }[]`. F3.2
    fills in the handlers.
- Declared in [`plugin.json`](../../plugins/parobek/.claude-plugin/plugin.json)
  under `mcpServers`, launched as `node ${CLAUDE_PLUGIN_ROOT}/mcp/server.mjs`.
- Handlers reuse `lib/provider.mjs` (`chat`) and `lib/config.mjs` (read config fresh
  per call — the server is long-lived, so re-read on each `tools/call` to pick up
  config edits).

## Files

- **Add:** `mcp/server.mjs`, `mcp/lib/rpc.mjs` (framing/dispatch helper, optional).
- **Change:** `plugin.json` (`mcpServers` block).
- **Reuse:** `lib/provider.mjs`, `lib/config.mjs`, `lib/brand.mjs`.

## Feasibility / risk

- Hand-rolling JSON-RPC is ~150 lines but must match the client's framing exactly;
  validate against a real Claude Code MCP handshake early.
- The server is a persistent process (unlike the one-shot command/hook scripts) — be
  careful to read config per call and never cache stale state.
- MCP servers load at session start — restart Claude Code after changing
  `plugin.json` or the server.

## Acceptance & verification

- Unit/integration: feed `initialize` then `tools/list` frames on stdin, assert a
  well-formed JSON-RPC response and that the advertised tool names match the registry.
- End-to-end: install the plugin, confirm the tools appear in Claude Code and a
  `tools/call` round-trips against `mock-server.mjs`.

## Depends on

being present so config helpers exist.
