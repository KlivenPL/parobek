# F5.3 — Auto pre-compact

- [ ] **Status:** not started
- **Tier:** 3 (Max) · **Phase:** NOW · **Epic:** 5

## Goal

When the live conversation crosses the top context-warning threshold, **automatically
prepare** a pending local summary, so the user only has to run `/clear` to apply it —
turning the existing warning into a ready-to-go action.

## Mechanism

- Extend the existing `UserPromptSubmit` flow
  ([`context-watch.mjs`](../../plugins/parobek/scripts/context-watch.mjs)) or add
  a sibling hook that self-gates on `featureEnabled(config, 'auto-precompact')`
  (level ≥ 3).
- When the top fraction is crossed, run the local summarization (reuse
  `local-compact.mjs`'s `summarize` + `writePendingSummary`) and emit a
  `systemMessage`: a summary is staged — run `/clear` to apply it.
- Respect the same debounce/flag mechanics as the warnings so it stages at most once
  per cycle.

## Feasibility / risk

- **A hook cannot run `/clear`** — it can only stage the pending summary; the user
  still applies it. The one-click "compact → clear" experience is the VS Code
  companion's job ([F6.5](../epic-6-vscode-companion/F6.5-tier-switcher-and-notifications.md)).
- Tier 3 because the next turn would otherwise be reasoning on a local summary
  (replacing the real context). Only auto-stage at `max`.
- Summarizing inside a `UserPromptSubmit` hook adds latency to that turn — keep the
  budget/threshold conservative, or spawn the summarization detached and only stage
  when ready.

## Files

- **Change:** `context-watch.mjs` (or add `scripts/auto-precompact.mjs` +
  `hooks.json`).
- **Reuse:** the `summarize` path and `writePendingSummary`/`clearWarnFlags` from
  `local-compact.mjs` / `state.mjs`, `lib/config.mjs` (gating).

## Acceptance & verification

- Hook test: simulate a transcript past the top fraction with `savingTier: max` →
  a pending summary is written and a "run /clear" `systemMessage` emitted; with a
  lower tier → only the normal warning (no staging).
