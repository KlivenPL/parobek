# F5.1 — File-reference pre-digest

- [ ] **Status:** not started
- **Tier:** 2 (Balanced) · **Phase:** NOW · **Epic:** 5

## Goal

When a prompt mentions a file path or URL, locally digest that target and inject the
digest as context, so Anthropic can often answer **without** a full `Read`/`WebFetch`
of the raw thing.

## Mechanism

- A `UserPromptSubmit` hook (`scripts/predigest.mjs`) self-gates on
  `featureEnabled(config, 'file-predigest')` (level ≥ 2).
- Detect references in the prompt: existing local file paths and `http(s)://` URLs.
- For each (bounded count/size), digest locally:
  - files via `local_summarize`/`local_read_digest` logic (reuse `lib/digest-prompt.mjs`);
  - URLs via a local fetch + summarize (skip if offline).
- Emit one branded `additionalContext` block with the digests, clearly labeled as a
  local pre-digest the model may rely on instead of re-reading.

## Feasibility / risk

- **Key constraint:** the hook only ADDS context; it cannot remove the user's text.
  So the token saving is **avoiding the later large tool-result**, not shrinking the
  prompt. Frame expectations accordingly.
- Tier 2 risk: the digest may omit a detail the raw file/URL contained → Anthropic
  works off partial input. Keep digests faithful and note they're lossy.
- Guard against huge fan-out (many references) — cap and prioritize.

## Files

## Acceptance & verification

- Hook test: prompt referencing a temp file → with `savingTier: safe` no output; with
  `balanced` an `additionalContext` digest of that file; assert oversize files are
  truncated/map-reduced and reference caps hold.
