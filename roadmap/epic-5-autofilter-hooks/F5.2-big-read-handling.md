# F5.2 — Big-read handling

- [ ] **Status:** not started
- **Tier:** 2 (nudge) / 3 (substitute) · **Phase:** NOW · **Epic:** 5

## Goal

Stop large raw `Read`/`Grep` results from flooding Anthropic's context. Two intensities:

- **Tier 2 — nudge:** guide Anthropic to *prefer* the local digest tools for large
  files (no interception).
- **Tier 3 — substitute:** actually return a local digest *instead of* the raw read.

## Mechanism

- **Nudge (tier 2):** delivered via [F1.4](../epic-1-tiered-savings/F1.4-tier-guidance-injection.md)
  guidance — "for files over N lines, use `local_read_digest`." No `PreToolUse` hook
  needed; purely advisory.
- **Substitute (tier 3):** a `PreToolUse` hook (`scripts/big-read.mjs`) on
  `Read`/`Grep`, self-gating on `featureEnabled(config, 'big-read-substitute')`
  (level ≥ 3):
  - estimate the target size; if under a threshold, allow the read untouched;
  - if over, **deny the call with feedback** — the feedback is the locally produced
    digest plus a clear label that this is a Parobek substitution and the raw file
    is available if needed.

## Feasibility / risk

- **The only lever** a `PreToolUse` hook has to substitute output is
  **deny-with-feedback**: the model sees a *blocked* `Read` and receives the digest as
  the denial reason. This is intentionally tier 3 and must be loudly labeled so the
  model understands it wasn't a real failure.
- Real quality risk: the model never sees the raw file for big reads. Threshold tuning
  matters; keep an escape hatch (e.g., a marker the model can use to force a raw read).
- Verify current Claude Code `PreToolUse` deny/feedback semantics against the official
  hooks documentation (https://code.claude.com/docs) before implementing — this is the
  most mechanism-sensitive feature in the roadmap.

## Files

- **Add:** `scripts/big-read.mjs`.
- **Change:** `hooks/hooks.json` (`PreToolUse` matcher for `Read`/`Grep`).
- **Reuse:** `lib/hookio.mjs`, `lib/config.mjs` (gating), `lib/provider.mjs`,

## Acceptance & verification

- Hook test: a `PreToolUse` `Read` of a large temp file → with level < 3 allowed
  untouched; with `max` denied-with-feedback carrying the labeled digest; a small file
  always allowed.
