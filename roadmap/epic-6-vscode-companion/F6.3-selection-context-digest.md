# F6.3 — Selection context digest

- [ ] **Status:** not started
- **Tier:** 2 (Balanced) · **Phase:** LATER (VS Code) · **Epic:** 6

## Goal

Let the user highlight code/text in the editor and, via a command/context-menu action,
locally digest the selection and drop the digest into the Claude conversation — so a
large highlighted region is summarized before it costs Anthropic tokens.

## Mechanism

- VS Code command "Parobek: Digest selection" (editor context menu + command
  palette).
- Read the active selection, call the digest logic (`lib/digest-prompt.mjs` +
  `provider.chat`), and insert the labeled digest into the prompt/conversation.
- Optional `local_extract`-style mode (selection → structured JSON) reusing the same
  tool family.

## Why LATER (not a CC hook)

- **CC hooks cannot see the editor selection** — only the extension has the editor
  API. This is the canonical example of a feature that *must* wait for the VS Code
  phase.

## Feasibility / risk

- Tier 2: the digest replaces the raw selection in what Anthropic sees → lossy. Make
  the insertion clearly labeled and easy to undo.
