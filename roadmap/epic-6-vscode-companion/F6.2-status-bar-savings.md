# F6.2 — Status bar savings meter

- [ ] **Status:** not started
- **Tier:** 0 (infra) · **Phase:** LATER (VS Code) · **Epic:** 6

## Goal

A persistent status-bar item showing the **active tier** and a running estimate of
**tokens saved** by Parobek this session, so the trade-off is always visible.

## Mechanism

- Status-bar item: `Parobek: Balanced · ~42k tok saved`.
- Tier comes from `resolveTier(config)` (F1.1).
- Savings estimate accumulates from the local operations performed: compactions,
  handoffs, pre-digests, big-read substitutions, and codesearch hits (each records an
  approximate "tokens that would otherwise have hit Anthropic"). Persist a per-session
  tally under `STATE_DIR` (reuse `state.mjs` IO + `cwdHash`).
- Reuse `lib/tokens.mjs` `estimate*` for the approximations.

## Feasibility / risk

- The savings number is an **estimate**, not metered Anthropic billing — label it as
  such. Keep the methodology documented so it's not mistaken for exact accounting.
- Needs the engine features to emit savings events; add a small shared
  `recordSavings(kind, tokens)` helper the CC scripts also call (so the meter works
  even when commands are run from the CLI).

## Depends on

[F1.1](../epic-1-tiered-savings/F1.1-tier-config-foundation.md); aggregates events
from Epics 2, 4, 5.
