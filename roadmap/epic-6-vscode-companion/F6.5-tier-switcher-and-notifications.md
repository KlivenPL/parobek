# F6.5 — Tier switcher + notifications

- [ ] **Status:** not started
- **Tier:** 0 (infra) · **Phase:** LATER (VS Code) · **Epic:** 6

## Goal

Make tier switching and the staged-compaction flow interactive: a one-click tier
picker and **actionable** notifications (e.g. "context is large — Compact & Clear?")
that the user can act on without typing commands.

## Mechanism

- **Tier switcher:** a status-bar/quick-pick UI that writes `config.savingTier` via
  `updateConfig` (the same write path as `/local-tier`, F1.2).
- **Actionable notifications:** subscribe to the engine's signals (the context-warning
  and auto-pre-compact staging from
  [F5.3](../epic-5-autofilter-hooks/F5.3-auto-pre-compact.md)) and surface a
  notification with buttons — e.g. "Compact now", and after staging, **"Clear to
  apply"** which the extension can execute.

## Why LATER (closes the NOW gaps)

- A CC hook **cannot run `/clear`** — so NOW the user must run it manually after
  staging. The extension can offer the one-click "Clear to apply", completing the
  auto-pre-compact loop F5.3 could only stage.
- Rich, interactive notifications need the extension host.

## Feasibility / risk

- Keep the CLI and UI in lock-step on config writes (always via `lib/config.mjs`).
- Don't surprise the user: any "Clear" action must be explicit (a button press), never
  automatic — same spirit as the five injection guards.
