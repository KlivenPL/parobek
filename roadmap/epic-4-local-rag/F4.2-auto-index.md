# F4.2 — Auto-index

- [ ] **Status:** not started
- **Tier:** 1 (Safe) · **Phase:** NOW · **Epic:** 4

## Goal

Keep an on-disk embeddings index of the repo **automatically**, with no command in the
normal path. Indexing just happens.

## Mechanism

- **Indexer** (`lib/index.mjs`): walk the repo from `cwd`, **respect `.gitignore`**
  (and skip binaries / oversize files), chunk each file (by symbol/line windows),
  `embed()` the chunks (F4.1), and store:
  - `index-<cwdHash>.json` under `STATE_DIR` (reuse `state.mjs` `cwdHash` +
    read/write helpers): `{ builtAt, model, files: { path: mtime }, chunks: [{ path, range, text, vector }] }`.
- **Auto-build triggers (no command):**
  1. **Lazy:** `local_codesearch` (F4.4) builds the index on first use if it's missing,
     and refreshes stale entries (compare stored `mtime` vs current) before searching.
  2. **Opportunistic:** a `SessionStart` hook spawns a **detached** refresh
     (`spawn(..., { detached: true, stdio: 'ignore' }).unref()`) so it never blocks
     session start. Self-gates on a local model + reachable server; silent on failure.
- **Incremental:** only re-embed files whose `mtime` changed since `builtAt`; drop
  entries for deleted files.

## Files

- **Add:** `lib/index.mjs` (walk/chunk/embed/store/search-core), `scripts/index-refresh.mjs`
  (detached entry point).
- **Change:** `hooks/hooks.json` (`SessionStart` → spawn `index-refresh.mjs`).
- **Reuse:** `lib/provider.mjs` (`embed`), `lib/state.mjs` (`cwdHash`, JSON IO),
  `lib/tokens.mjs` (chunk sizing).

## Feasibility / risk

- One-shot + detached refresh is the **NOW** ceiling; true continuous file-watching
  needs a daemon → deferred to [F6.4](../epic-6-vscode-companion/F6.4-filewatch-indexer.md).
- Large repos: cap total chunks / time per refresh; the lazy path always closes the
  gap before a search.
- Embedding cost is local (0 Anthropic tokens) but uses local RAM/CPU — keep the
  detached refresh low-priority and interruptible.

## Acceptance & verification

- Build the index against a temp repo (with a `.gitignore`) using `mock-server.mjs`
  `/embeddings`; assert ignored files are excluded, `mtime` change re-embeds only the
  changed file, and deleted files are pruned.
