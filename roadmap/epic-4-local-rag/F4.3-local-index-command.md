# F4.3 — `/local-index status|rebuild`

- [ ] **Status:** not started
- **Tier:** 1 (Safe) · **Phase:** NOW · **Epic:** 4

## Goal

A command to **inspect** the auto-built index or **force** a rebuild — never the
normal way to index (that's automatic, F4.2). It exists for visibility and manual
override only.

## Mechanism

```
/local-index            # alias of status
/local-index status     # show index age, indexed-file count, chunk/vector count,
                        #   embedding model, staleness (files changed since builtAt)
/local-index rebuild    # discard and rebuild the index from scratch (foreground,
                        #   with progress)
```

- `scripts/local-index.mjs`: read arg; `status` prints metadata from
  `index-<cwdHash>.json`; `rebuild` calls the F4.2 indexer with a force flag and
  prints a summary (files, chunks, elapsed, 0 Anthropic tokens).
- `.md` wrapper, `allowed-tools: Bash(node:*)`.

## Files

- **Add:** `commands/local-index.md`, `scripts/local-index.mjs`.
- **Reuse:** `lib/index.mjs` (F4.2), `lib/state.mjs`, `lib/brand.mjs`,
  `lib/config.mjs` (`warnIfTierExceeds` — no-op at tier 1).

## Feasibility / risk

- Keep the contract clear in help text: indexing is automatic; this command only
  inspects or forces. Avoid implying you must run it to enable search.

## Acceptance & verification

- Integration: with an existing index, `status` prints the expected counts; after a
  file change, `status` reports staleness; `rebuild` produces a fresh `builtAt`.
