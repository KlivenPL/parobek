# F4.1 — Provider embeddings

- [ ] **Status:** not started
- **Tier:** 1 (Safe) · **Phase:** NOW · **Epic:** 4

## Goal

Add an `embed()` capability to the provider abstraction so the index and search can
turn text into vectors via the local server's `/embeddings` endpoint.

## Mechanism

- Extend the provider facade
  [`provider.mjs`](../../plugins/parobek/scripts/lib/provider.mjs) with
  `export const embed = (config, texts, opts) => getProvider(config).embed(config, texts, opts)`.
- Implement `embed(config, texts)` in `providers/openai.mjs` (the vendor-neutral base)
  as a POST to `{endpoint}/embeddings` with `{ model, input: texts }`, returning
  `number[][]`. `lmstudio.mjs` / `ollama.mjs` inherit unless their route differs.
- Config: add an optional `embeddingModel` (falls back to a sensible default or
  errors clearly if the server has no embedding model). Embedding models are separate
  from the chat model.
- Reuse the existing error handling (`LocalModelError`) and timeout patterns from the
  chat path.

## Files

- **Change:** `lib/provider.mjs`, `lib/providers/openai.mjs` (+ `lmstudio.mjs` /
  `ollama.mjs` only if their `/embeddings` differs).
- **Change:** `lib/config.mjs` (`DEFAULT_CONFIG.embeddingModel`, summary line).

## Feasibility / risk

- Most OpenAI-compatible local servers expose `/embeddings`, but not all loaded models
  support it — fail with a clear, branded message pointing to load an embedding model.
- Batch inputs to respect server limits.

## Acceptance & verification

- Add an `/embeddings` route to `mock-server.mjs` returning deterministic vectors;
  unit-test `embed()` returns the right shape and handles a missing-model error
  cleanly.

## Depends on

Nothing; prerequisite for [F4.2](F4.2-auto-index.md) and
[F4.4](F4.4-local-codesearch-tool.md).
