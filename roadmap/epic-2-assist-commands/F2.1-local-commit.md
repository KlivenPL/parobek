# F2.1 — `/local-commit`

- [x] **Status:** done
- **Tier:** 1 (Safe) · **Phase:** NOW · **Epic:** 2

## Goal

Draft a Conventional Commit message from the staged diff using the local model, so
the user/Claude never spends Anthropic tokens writing routine commit messages.

**Tier 1 rationale:** the output is a *reviewable draft*; the diff stays fully visible
and the user approves before anything is committed. It only ADDS a cheaper path.

## Mechanism

```
/local-commit                  # draft from the staged diff + conversation context
/local-commit <focus>          # optional focus hint (scope, "explain why", etc.)
/local-commit --short|--full   # override verbosity for this run
/local-commit --no-context     # ignore the transcript, draft from the diff only
/local-commit apply            # draft, then run the commit with the message
```

- `scripts/local-commit.mjs`:
  1. Staged-diff preflight: read `git diff --staged` + `git status --short`. If nothing
     staged → say so, suggest `git add`, stop. Then model preflight: selected +
     reachable (`ping`).
  2. **Verbosity** (`short` default = subject + tight bullets, `full` = subject + prose
     body explaining *why*) resolves from `--short`/`--full` else `config.commitVerbosity`.
  3. **Context** (default on, `config.commitContext`, off with `--no-context`): feed the
     recorded session transcript (`readSessionRecord` + `parseTranscript`) so the message
     captures intent. The **diff is the source of truth for *what* changed**; context only
     informs *why*.
  4. Send to the local model with a Conventional-Commit prompt (`lib/commit-prompt.mjs`):
     subject ≤ ~72 chars, imperative, `type(scope): summary`. Pass the focus hint if given.
  5. Print the proposed message (do **not** auto-commit unless `apply`); `apply` writes
     the message to a temp file and runs `git commit -F`.
  6. Large diffs: map-reduce fallback (split per-file → chunk → digest → compose) so a
     huge diff doesn't blow the local context. The diff stays authoritative — context is
     trimmed (oldest first) before the diff is ever reduced.
- `.md` wrapper with `allowed-tools: Bash(node:*)` (the script runs the commit itself, so
  the agent never needs git → the 0-token guarantee holds).

  (`parseTranscript`) for context, `lib/config.mjs` (`warnIfTierExceeds` — no-op at tier 1).

## Feasibility / risk

None to Anthropic quality (draft only). Only risk is a low-quality message, which the
user sees and can edit. `apply` is the only state-changing path and is opt-in.

## Acceptance & verification

- Integration via `helpers/run.mjs` against a temp git repo with a staged change +
  `mock-server.mjs`: assert a Conventional-Commit-shaped message is printed and that
  bare `/local-commit` does **not** create a commit; `apply` does.
- Smoke: stage a change, `node scripts/local-commit.mjs`.
