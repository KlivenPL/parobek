# F2.2 — `/local-ask`

- [ ] **Status:** not started
- **Tier:** 3 (Max) · **Phase:** NOW · **Epic:** 2

## Goal

Answer a throwaway question **entirely with the local model** — 0 Anthropic tokens,
Anthropic never invoked. For quick lookups ("what does this regex do", "git command
to undo a merge") where Claude-grade reasoning is overkill.

**Tier 3 rationale:** this fully REPLACES Anthropic's reasoning for the answer. The
configured-tier warning (F1.3) fires whenever `savingTier < max`.

## Mechanism

```
/local-ask <question>
/local-ask --file <path> <question>   # ground the answer on a local file
```

- `scripts/local-ask.mjs`:
  1. `warnIfTierExceeds` (tier 3) — warn unless the user is on `max`.
  2. Preflight: model selected + reachable.
  3. Optional `--file`: read + truncate it (reuse the `transcript.mjs` truncation
     style / `MAX_BLOCK_CHARS`) and prepend as context.
  4. `chat` with a concise-answer system prompt (`lib/ask-prompt.mjs`).
  5. Print the answer under a loud banner so it's never mistaken for Claude:

     > `[Parobek] answered locally by <model> — NOT Claude. Verify if critical.`

- `.md` wrapper, `allowed-tools: Bash(node:*)`. No transcript needed.

## Files

- **Add:** `commands/local-ask.md`, `scripts/local-ask.mjs`, `lib/ask-prompt.mjs`.
- **Reuse:** `lib/provider.mjs`, `lib/brand.mjs`, `lib/transcript.mjs` (truncation),
  `lib/config.mjs` (`warnIfTierExceeds`).

## Feasibility / risk

- Real quality risk by design — the banner makes the source explicit, and the tier
  warning makes overruns of the configured floor explicit. That's the whole point of
  tier 3.
- Keep the answer self-contained; do not let it silently feed back into Anthropic
  context as authoritative (it's printed for the user).

## Acceptance & verification

- Integration via `helpers/run.mjs` + `mock-server.mjs`: assert the banner is present
  and the mock chat route was hit; with `savingTier: safe` assert the tier warning is
  also printed.
- Smoke: `node scripts/local-ask.mjs "what does chmod 755 mean"`.
