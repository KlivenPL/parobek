# F1.4 — Tier-guidance injection

- [x] **Status:** done (shipped alongside [F3.2](../epic-3-mcp-toolbox/F3.2-digest-tools.md) — the tools it nudges toward)
- **Tier:** 2 (Balanced) · **Phase:** NOW · **Epic:** 1

## Goal

At Balanced and above, steer Anthropic toward the local MCP tools so it routes large
reads through `local_read_digest` / `local_summarize` instead of raw `Read`. This is
how a tier "leans on" the local model for MCP usage: stronger guidance, not new tools.

## Mechanism

- A `SessionStart` hook (`scripts/tier-guidance.mjs`) self-gates on
  `featureEnabled(config, 'tier-guidance')` (level ≥ 2). When active it emits
  `hookSpecificOutput.additionalContext` with a short, branded note, e.g.:

  > `[Parobek] Savings tier: Balanced. Prefer local_read_digest / local_summarize over a raw Read for large files; use local_codesearch before reading many files.`

- At `max`, the note is stronger (also references the big-read substitution behavior
  so the model understands why some reads come back as digests).
- Registered in `hooks.json` under the existing `SessionStart` matcher (added after
  `record-session` / `inject-summary`).

## Files

- **Add:** `scripts/tier-guidance.mjs`.
- **Change:** `hooks/hooks.json` (add to `SessionStart`).
- **Reuse:** `lib/hookio.mjs` (`readHookInput`/`emit`), `lib/config.mjs`
  (`resolveTier`/`featureEnabled`), `lib/brand.mjs`.

## Feasibility / risk

- Clean: `SessionStart` `additionalContext` is a supported, additive channel.
- Low value until Epic 3 tools exist — until then it nudges toward tools that aren't
  there. Land it with or after Epic 3.
- Remember: hooks load at session start — restart Claude Code after editing
  `hooks.json`.

## Acceptance & verification

- Hook test: feed a `SessionStart` event with `savingTier: safe` → no output;
  with `balanced` → `additionalContext` contains the guidance; with `max` → stronger
  variant.
