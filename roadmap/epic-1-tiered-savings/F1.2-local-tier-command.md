# F1.2 — `/local-tier` command

- [x] **Status:** done
- **Tier:** 0 (infra) · **Phase:** NOW · **Epic:** 1

## Goal

A slash command to inspect and switch the active savings tier — the parallel of
`/local-model preset <name>`.

## Mechanism

```
/local-tier                 # alias of status
/local-tier status          # show current tier + level + which features it activates
/local-tier safe|balanced|max   # switch tier (writes config.savingTier)
```

- Thin `.md` wrapper (`commands/local-tier.md`) running
  `node ${CLAUDE_PLUGIN_ROOT}/scripts/local-tier.mjs "$ARGUMENTS"`, with
  `allowed-tools: Bash(node:*)` and `disable-model-invocation: true` (same shape as
  the other command wrappers).
- `scripts/local-tier.mjs`: read arg; on a valid tier key call `updateConfig({ savingTier })`
  and print the resolved tier; on `status`/empty print `resolveTier` + the feature
  list from `TIER_PRESETS`; on an unknown arg print the valid keys.
- Reuse `resolveTier`, `updateConfig`, `configSummaryLines` (F1.1) and `brand.say`.

## Files

- **Add:** `commands/local-tier.md`, `scripts/local-tier.mjs`.
- **Reuse:** `lib/config.mjs` (F1.1 helpers), `lib/brand.mjs`.

## Feasibility / risk

None — config write only. Validate the tier key against `config.tiers` (built-ins +
custom) so a user-defined tier is also selectable.

## Acceptance & verification

- Integration test via `helpers/run.mjs`: spawn with `balanced`, then `status`,
  assert the printed level/name and that `config.json` was updated.
- Smoke: `/local-tier max` then `/local-tier status`.
