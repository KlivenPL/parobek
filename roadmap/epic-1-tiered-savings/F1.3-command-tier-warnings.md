# F1.3 — Command-tier warnings

- [x] **Status:** done
- **Tier:** 0 (infra) · **Phase:** NOW · **Epic:** 1

## Goal

Commands are never blocked by the configured tier, but when a command operates at a
**higher** tier than configured, it must warn that quality may fall below the user's
intended floor.

## Mechanism

- A `COMMAND_TIER` map (in `lib/config.mjs` or a small `lib/tier.mjs`): command id →
  tier level. e.g. `{ 'local-commit': 1, 'local-handoff': 2, 'local-ask': 3, 'local-compact': 2 }`.
- A shared helper `warnIfTierExceeds(config, commandId)`: if
  `commandTierExceeds(config, COMMAND_TIER[commandId])`, print via `brand.say`:

  > `[Parobek] /local-ask runs at tier 3 (Max), above your configured tier 1 (Safe) — output quality may fall below what you set.`

  Returns nothing; no-op when within the configured tier.
- Every `/local-*` command calls `warnIfTierExceeds` once, early, before its work.

## Files

- **Add:** `warnIfTierExceeds` (in `lib/config.mjs` or `lib/tier.mjs`) + `COMMAND_TIER`.
- **Change:** each command script (`local-commit.mjs`, `local-ask.mjs`,
  `local-handoff.mjs`, optionally `local-compact.mjs`) to call it.

## Feasibility / risk

None. The warning is informational and printed to command stdout (already the
plugin's standard surface).

## Acceptance & verification

- Unit test the helper: tier 3 command under `safe` config → warning string emitted;
  under `max` config → silent.
- Integration: run `/local-ask` (tier 3) with `savingTier: safe` and assert the
  warning line is present in stdout.
